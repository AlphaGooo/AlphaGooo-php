
***~~程序执行期望能捕抓到异常信息，但是不希望中断进程，采取try监控进程。~~***

```
<?php
    // 1. 通过try去监控一个业务，期间可以捕抓异常，可以控制终止或捕抓记录
    try {
        // 监控整个业务逻辑的过程，可以捕抓过程中的所有异常
        
        #============== 具体业务逻辑 开始 ================#
        echo php_info();
        #============== 具体业务逻辑 结束 ================#
        
    } catch (\Exception $e) {
        // 2. 一旦业务逻辑出现异常或者报错，都会进入 catch 模块，这里可以根据需求抛出异常或者采取其他操作
        
        throw new $e;
    }
?>
```
